## Docs

## SharePoint

### Issue Tracker

1) Atualizar as Issues do cliente:
	1) Com o status correto (`Ready for testing` ou `Completed`)
	2) Com a versão na qual a issue está sendo entregue (`Deliverable`)
	3) Com a `Resolution Date` quando for o caso

### Software Download

1) Manter apenas a última versão gerada 
	1) Incluir a data de liberação da versão

### Deliverables

1) Ajustar o status da versão
2) Ajustar a data de release da versão
3) Criar a próxima versão (caso não exista), não incluir data de release
